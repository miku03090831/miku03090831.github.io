使用telnet模拟http请求
发布于 2020-02-15   更新于 2020-02-15 应用层 热度 0 喜欢 0

去年双11的时候，趁着书打折，几乎是半价入了几本好书。现在想起来看。这几天在看《计算机网络-自顶向下方法》。这本书的讲解非常细致，并且鼓励动手实践。我在看到http报文的时候，按照书上说的试着用telnet模拟了一个请求报文，不过中间还是遇到了些困难，书上没有提到。特此写一篇就当做踩坑经验了。

首先要开启telnet

控制面板->程序->启用或关闭windows功能

找到telnet client，打上勾，开启telnet服务

其次打开cmd，用telnet登录服务器

以书上例子为例，输入telnet gaia.cs.umass.edu 80，敲下回车，即可连接到目标服务器的80端口

第一个坑

我怎么输入的东西在屏幕上不回显呢？

只有一个闪烁的光标，然而无论你输入什么屏幕上都不会回显，而只要你一敲回车，就会收到http响应报文（400 bad request)。因为你输了乱七八糟的东西，但是看不到，发给服务器后无法识别，响应报文中的状态码就是400.

解决方法：按下ctrl+]键，即可使输入回显，如下图所示：

第二个坑

http请求报文里面要换行怎么办呢？我一敲回车报文就发出去了

http报文通常不止一行（第一行是request line，后面还要有header line），而这个换行怎么办呢。。。书上明明说的是最后一行要连敲两个回车（换行加回车）作为结束的

解决方法：在输入之前先按下回车，就会跳出一个全新的窗口，可以在新的窗口里面输入完整的报文。输入完毕后，连敲两个回车，就可以收到正确的响应报文了。

另，时间长了可能会显示遗失对主机的连接。可能来不及输入就已经断开连接了。所以可以提前把报文写在记事本里，然后复制后，在命令行里面点击右键就是粘贴啦

结果如下图，我们可以看到状态码是200，是对哒：

然后呢，我们的任务就完成啦！成功用telnet发送了一个http请求报文并且得到了服务器的响应报文
