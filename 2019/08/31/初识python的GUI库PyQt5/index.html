初识python的GUI库PyQt5，记一些踩过的坑
发布于 2019-08-31   更新于 2019-09-02 python 热度 0 喜欢 0

简短总结本文内容：我费了好大劲终于配好了PyQt5的环境，然后试着跑了几段代码，能用了！

此处打算放个目录（因为前面废话有点多）

学习了一段时间的python，也想试着去做一点东西了。一般来讲做一个小项目的话也总离不开图形界面的开发（也就是常说的GUI）。

而python自己是带了图形库的，tkinter。这个库只能说是中规中矩，优点就是它身为python的原装标准库，python对它的支持肯定是很好的，而且使用的人也比较多。除此之外，它也很简单，轻量。但是这也是小小的不足，因为轻量所以不够强大，写一个自己凑合用的是足够了，但是如果想写一个美观又吸引人的界面出来，tkinter有点乏力。

这时候我们就要去寻找一个其他的图形库。这里强烈推荐PyQt5，因为它功能强大却不繁琐，最最重要的一点是它的使用群体庞大。这对于程序员是非常重要的，因为用的人多代表着你遇见的问题很可能别人也遇见过，并且已经提供了解决方案。程序员最好还是不要去使用那些非主流的东西，因为你可能会遇见无尽的问题，这个时候你就真切的体会到什么叫孤儿。

其实我有一点GUI经验，不过是用c语言搭配一个非常有年头的图形库写的（其实都是队友写的），也对于GUI有一定的认识。以我的理解，GUI就是用代码来调用图形库，然后图形库再来调用操作系统的接口实现图形界面（我们的操作系统也可以说是GUI图形界面，与之相对的就是命令行界面）。

我上面说了那么多PyQt5的好话，一开始也当然是尝试过的，不过很遗憾由于我的python环境比较乱，系统的环境变量也有点乱套，所以总是遇到一些意料之外的错误。（因为之前自己学python的时候先装了一个官方的IDLE，后面又试过用vs2017写python又装了一个解释器，最后有门课程又用到了anaconda）。那么不说废话了，先来说说PyQt5的环境配置（即如何安装PyQt5）。

环境配置

首先这里推荐anaconda+vscode的配置（习惯问题而已），anaconda你可以理解为一个加强版的python解释器，他自己天生就附带了超多的扩展包和很多工具，下载链接,挑个新一点的版本。当然，用官方的标准版python也足够了，只是要自己去下载pip（用pip来安装并管理模块）。

至于vscode，宇宙第一编辑器不用我多说，简单查一下如何用vscode写python就可以了。其实pycharm也可以，只是我没用过就不说了。

特别强调，如果之前有python环境而且比较乱的话，建议删干净，删不干净大不了重做系统。。。（没办法，年少无知的时候犯下的错，把电脑搞得乱七八糟的，只能重做）

在vscode新建一个py文件，然后在终端输入pip install sip(注意先安装这个)，不出意外都会成功，然后输入pip install pyqt5(包的名字不区分大小写)，最后再输入pip install pyqt5-tools

最后为了测试是否成功，在py文件中输入代码from PyQt5.QtWidgets import QWidget, QPushButton, QApplication，测试文件能否运行。
