何为闭包

    说到闭包，我第一个问题就是什么是闭包。离散的课上倒是讲过关系的闭包，什么自反闭包，传递闭包之类的，不过那个跟编程语言里面的闭包现在来看好像不太相关。

    那么我看了很多篇文章，自己总结出来的闭包的定义就是，有一个嵌套定义的函数，外层函数的返回值是内层函数，并且内层函数用到了外层函数的local变量（即局部变量，也有的人叫它free variable），特别的，此变量不能是全局变量。

If a name is bound in a block, it is a local variable of that block. If a name is bound at the module level, it is a global variable. (The variables of the module code block are local and global.) If a variable is used in a code block but not defined there, it is a free variable.

如果一个变量被限制在一个代码块（这里可以理解为一个函数）内，它是这个代码块的local变量（局部变量）。如果它被限制在module级别的块内，那它也是一个全局变量（同时也是这个块的local变量）。

如果一个变量在某代码块内被使用，但是它的定义不再这里，它叫做free variable

特此说明，free variable，free变量，自由变量是一个东西

​ local variable，local变量，局部（本地）变量是一个东西

​ global variable，global变量，全局变量是一个东西

以免有些人看糊涂了

  • 闭包中引用的free变量，是跟每个具体的闭包挂钩的，不同的实例，free变量不会互相影响
  • 对于同一个闭包的实例，free变量的值是可以被记住的。对free变量的值修改，会一直保留下去

变量的作用域

我看了很多文章里面介绍了各种关乎闭包的经典错误，我认为都可归结于对变量的作用域没有理解到位（由于python是一门动态语言，所以也是个麻烦事）。在静态语言，如C/C++中，变量需要先声明，而python中就可能出现比较神奇的情况。

乍一看，好像没啥毛病，inner()函数试图引用外层定义的变量a，然后把a+1，再打印出来。但是当你试图去执行这段代码，解释器会给你报错local variable ‘a’ referenced before assignment。说什么呢，local变量引用出错了，你没给它赋值。咋回事啊，小老弟，我用的外层函数的free变量，咋还没赋值呢

    是这样的，注意报错里面写的是local变量a，说明解释器在这里把a当做是inner函数新设出来的一个变量，和外层函数的a不是一个东西。python规则规定，在函数里面出现在等号左边的是一个局部变量。执行f=outer()的时候，解释器会分析闭包里面的局部变量。也就是说，此时解释器认为，inner()里面的a是局部变量。这样就很好懂了，a是一个新的变量，没有被赋初值，然后直接执行a = a + 1，肯定会报错。

解决的方法呢，就是用nonlocal声明a，告诉解释器，这个a不是本地变量，而是一个自由变量（它的定义或声明在外层函数里）。另外需注意，nonlocal不能把函数内的变量指定为全局变量，只能指定为自由变量。

关于变量的作用域，在本篇里不再深入讨论。很多出错的原因，都是因为忘记了这句话在函数里面出现在等号左边的是一个局部变量。给个传送门，我看到的一篇讲的还不错的关于作用域的文章。
