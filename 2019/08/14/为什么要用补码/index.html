为什么要用补码
发布于 2019-08-14   更新于 2019-08-15 计算机基础 热度 0 喜欢 0

我们为什么需要补码

大一上翻书的时候，看到书上讲了原码，反码，和补码什么的，当时看了不是很懂，不明白为什么要这么费劲，觉得都用原码表示挺方便的。老师上课也没有细讲，懒得去吐槽那个老师了。昨天晚上看到篇文章，才稍微想明白了为什么要用补码去表示负数

另注明：本文中有些稍微混淆了负数与减法的概念，希望读者能够体会到我的意思，不要过于纠结，因为加上一个负数，就等于减去他的绝对值，所以说负数的问题和减法的问题是一个问题。

基本概念

为了简单起见，我们用8位机来说明

原码：我们如果用八位来表示一个数字的话，可以表示256个数（2的8次幂）。如何表示数的正负？由最高位数字决定。最高位为0，则是正数。最高位为1，则是负数。

补码：在表示负数时，将负数的原码作如下处理：最高位（符号位）不变（也就是为1），后面所有位取反。这时得到的叫反码。对反码+1，则得到补码。注意，补码的概念是针对负数的。

也就是说，在计算机里面，非负数都是用原码表示的。而负数，则是用补码表示的。那么为什么要用两个标准来表示呢？我最开始也觉得很麻烦

从二进制与门电路说起

无论是原码，还是补码，都是在用二进制来表示数字。二进制，0和1，与布尔值的false和true对应。计算机里用二进制存数字，那么做加法时，也用的是二进制。二进制的加法怎么写呢，来列个竖式

对每一位依次操作，从右向左开始运算，1+1=2，二进制里面每一位最大是1，所以得到2时，要把这一位记为0，向下一位进1。剩下的，0+0=0,0+1=1，正常算就可以。看一下我们列的竖式，按照二进制加法的法则，确实得到了正确答案8。

那么这些加法在计算机里面是如何实现的呢，也很简单，都是由基本的门电路来实现的（由异或门和与门组成半加器，两个半加器加上一个或门得到一个全加器，好多个全加器加起来就得到了加法器，这都不重要，总之就是做加法很方便）。

并且，我们不想做减法，只想做加法。如果需要减一个数，我们会加上它的相反数。因为如果要做减法的话，还要另外用一套逻辑。上面加法的逻辑比较简单，我们想简单一点。

这就是为什么存负数需要用补码，一切都是为了方便，做加法的那套逻辑电路也能用来做减法。

想让计算机只做加法

下面我讲说明补码是怎么来的，这是我个人的理解，不过我觉得我说的应该没毛病，如有错误还欢迎指正。

我自认为我下面说的可以解释通补码是怎么来的，至于文章一开头讲的，什么取反加一，我认为那是先有了补码的定义，然后人们为了得到补码，总结出来的方法。

假设我们就不知道什么补码。我们想表示所有的数，但是不想用原码那套方法来表示负数。我们先表示正数（和零），就直接把一个数化为二进制。那么负数呢，我们的想法，是用0（00000000），去减负数的相反数。例如，我们想得到-8，则用下面的式子

注意了，00000000当然是不够减的，需要向假想的“第九位”借1，所以实质上是100000000，减去一个正数，得到了这个数的相反数。

我们姑且先把这个结果叫做“负X”。这个“负X”是什么呢，是我们在脑子里做了一个减法，是用00000000（或者说是100000000）减去一个正数X，得到的。我们让计算机加上这个“负X”，计算机天真的就真去加了这个”负X”，实质上相当于减去了X（加0，减X）。计算机没有做减法，但是我们用计算机的规则，人脑做了一个减法，并且把减法的结果给了计算机。

我们可以这样想，每当计算机要做一个减法时，比如减去“X”，我们都可以用脑子去算一下0-X，把得到的结果记为“负X”，交给计算机，告诉他你只要加上这个东西就可以了，然后计算机按照你的指令做了个加法，得到了一开始的减法的结果。

补码是怎么来的

那每次算减法我们都要人工的做一个减法，也不是个事啊，所以我们在想，有没有什么办法能让计算机试着去做减法。我们仔细看一下人脑做的那个减法，1000000000-X，也可以写成11111111+1-X，即（11111111-X）+1，而11111111-X，很巧的是，他正好对应计算机里面的一个逻辑运算“取反”。它相当于把X里面的每一位，全部取反。

这么看来，我们梳理一下。我们口中的“负X”，从一开始的00000000-X，变成了100000000-X，然后变成了（11111111-X）+1，又变成了**对 X 按位取反，再加一**。当然了，我们首先确定的是X是一个正数，所以X的最高位是0，取反后变成了1。这个操作，是否有点眼熟呢？怎么和一开始说的补码一模一样啊，来看看补码是怎么说的。负数的补码：符号位不变（是1），其余位按位取反，最后加一。现在快赶紧一拍大腿，仰天长叹：“原来我一开始口中的“负X”，其实就是补码啊！”

或许你可能觉得此时信息量有点大，我们不妨来回顾一下我们的历程：

  • 因为用原码来表示正负数的话，正数和负数不能用一套运算的电路，结果是错的
  • 按理来说减去一个数，等于加上这个数的相反数，但是我们的例子告诉我们，如果用原码的话，得到的答案是错的
  • 此时，如果我们仍选择用原码，就要为减法设计自己的逻辑（好麻烦根本不想干），所以我们选择用补码
  • 补码表示负数的目的，就是让负数和正数相加，能得到正确的结果
  • 为了正确的结果，我们决定用人脑来做减法（被零减），并且把这个结果告诉计算机，这样计算机执行了加法的指令，却做了我们口中的减法
  • 后来我们想个法子让计算机来代替人脑“做减法”（实质上并没有做减法，而是通过按位取反 并加1的操作，得到了和人脑做减法相同的结果）
  • 我们管这个“减法“的结果叫做补码，并且总结出了如何从原码得到补码的规则（写在文章开头）

最后有一个精华的等式：11111111-正数+1=100000000-正数

等式左侧，是我们取补码的操作。想知道某个负数如何用补码表示，只要把负数的绝对值，带入左侧式子

我们由左侧的式子，去找寻它的现实意义，就是符号位为1，剩下位取反再加一

等式右侧，是补码的原理，是关于为什么加上补码就是在做减法。

而这个等式，从左往右解读，就是：我们按照左侧表示的操作，会得到一个东西，叫补码。这个补码呢，表示一个负数，加上它相当于减去一个正数。

小结

一开始人们没想过那么多，只想着用原码的方式表示，因为这是最好想到的，很自然会想到拿一个位来表示符号。但是后来为了让计算机只做加法，人们把减法做在了脑子里，直接把负数以补码的形式放在计算机里，计算机就可以只做加法了。一开始我们叙述的是得到补码的方法，补码的原理写在了中间。如果有点糊涂的话，可以去回顾一下
