python面向对象的一些技巧
发布于 2019-08-28   更新于 2019-08-28 python 热度 0 喜欢 0

这几天也是上网看了一些东西，感觉慢慢深入了解了面向对象的一些思想。之前只是知道什么是面向对象，但仅限于知道一些最基本的东西。这篇文章不适合于第一次听说面向对象的朋友，而是适合面向对象刚刚入门，对于类和对象，继承等基本概念有所了解的朋友来阅读。在看了这篇文章后，我相信各位都能掌握一些优雅的技巧。


首先清楚一点，对象内部的数据分两种，一种是变量类型，叫属性。另一种是函数类型，叫方法。

面向对象的要点

面向对象的三个要点：封装，继承，多态。我们这篇文章只介绍封装用到的技巧，让你的封装可以写的更加优雅。

封装

何为封装？面向对象的思想就是把数据（属性和方法）都封装在对象中，然后提供一些简单的接口，让使用者可以方便调用，却不必关心如何实现（最好也不能轻易修改对象的数据）。简单的来说，你只需要让用户知道应该如何使用，而不必让他们操心用户的指令是如何被处理，进而达到他们的目的的。

有点神奇。我们把对象的属性名字前面加了两个下划线后，后面就不能直接给它赋值了（如果直接访问也会报错）。这样似乎就放置了从外部修改对象的属性值。我们有点好奇，因为我们赋值的时候并没有得到报错，但是却也没有赋值成功。这样的结果有点奇怪，不妨用dir()函数，来看一下s的属性和方法。

我们得到了一个列表。看上去有点乱，但仔细一看能发现几个眼熟的名字‘_Student__age’，’_Student__name’，还有‘__name’。我们去尝试打印这几个变量，得到结果后就明白了。所谓的加双下划线阻止访问，不过是自动帮你改了一下属性的名字。在前面加上了单下划线+类名。也就是说我们定义的时候因为加上了双下划线，所以__name在外部访问的时候名字变成了_Student__name，也就不难理解为什么我们复制不成功却没报错了，解释其认为我们是新加了一个属性。

那这样看来，有点意思。阻止访问好像只是让你更加麻烦了一点，但并没有彻底地阻止你。事实上，上面讲的换名规则，不一定所有的解释器都遵守。个别解释器可能有自己的改名的方法，不过你依然可以通过dir()函数来看看名字是怎么改的。python的这个做法很有意思，它不彻底阻止你，只是告诉你最好不要这样做，并给你找了点麻烦。这是因为，程序员不希望完全的封闭。而且，程序员要为自己的行为负责，设计者觉得程序员不会无缘无故去访问私有属性（增加了一步防止误操作）。

更进一步的封装

上面已经讲了，初步的访问限制。而其实我们知道，这并不是严格限制的，它基于程序员的自觉。而我们有时候，加双下划线会确实导致一些预想之外的访问限制问题。所以为了避免麻烦，我们一般加单下划线。单下划线的意思是，这个变量我不希望你从外部直接访问。当然这并不受语法的保护，只是单纯的希望你能够自觉不去访问它。

而我们不希望直接去访问，当然会提供接口啦，而且是很方便的接口。这里就要用到装饰器了。这里你不需要了解装饰器的语法，只需要记住如何使用，如果想深入了解，给你个传送门去学习吧。
