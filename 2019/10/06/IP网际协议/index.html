IP网际协议
发布于 2019-10-06   更新于 2019-10-07 TCP/IP 热度 0 喜欢 0

背景：TCP/IP协议定义了在因特网上传输的包，称为IP数据报

IP协议TCP/IP模型中，位于网络层的协议，是整个协议族的核心。所有的TCP，UDP以及ICMP等数据都一IP数据报的格式传输。IP协议提供的数据报传输服务的特点是：不可靠，无连接

  • 不可靠：IP协议不保证能把数据正确的送到目的地，只提供传输服务。所有的可靠性都由上层协议来确保。如果出现错误，IP协议会丢弃出错的数据报
  • 无连接：IP协议对每个数据报的处理是独立的，也就是说接收方不用按照发送方发送的顺序来接受。

下面来看看IP数据报的具体格式

IP数据报

IP数据报由两部分组成，分别是首部和数据。

其中首部就是在网络层加上去的，而首部又分为两部分。前一部分是固定字段，20字节，大家都有。后一部分是可选字段，长度可变

其中，这里说的位，都是二进制位，只能是0或1

固定部分

固定部分有很多部分组成

  1. 协议版本：占4位，目前只有IPV4和IPV6两个版本，且主流仍使用IPV4版本。0100表示IPV4，0110表示IPV6
  2. 首部长度：占4位。特别注意，首部长度的数值，单位是32位（也就是四个字节）。如果IP数据报中，首部长度是这么写的0101，它的十进制值为5，则代表首部长度有5*32位，也就是5*4=20个字节（正好是固定部分的长度，代表这个IP数据报的首部没有可选字段）。其实，20字节也是最常见的IP数据报首部长度。
    首部长度这里最大值是15，则说明真正的首部长度最长也只有15*4=60字节
  3. 区分服务（以前叫做服务类型）：占8位，分别有：最小时延、最大吞吐量、最高可靠性、最小花费 4 种服务，如下图所示。4 个标识位只能有一个被置为 1。
  4. 总长度：占16位，这个长度指的是首部和数据长度之和也就是IP数据报的长度，单位是字节。也就是说，一个IP数据报，算上首部和数据，一共最多长度是2^16-1=65535字节

不过上一篇里面讲了，链路层是有MTU（最大传输单元）的，比MTU大的IP数据报都要被分段才能传到链路层，分成好多个段的数据报，每一段都比MTU小。

到这里我们先停一下，回到最开始我们说了IP协议是无连接的，也就是说接收的时候不必按照发送的顺序。那问题出现了，我如果把原本一整个大的IP数据报分段了，分成好多个小的数据报发出去，然后他接收的顺序又不一定按照我发送的顺序，那他收到一堆数据报，他怎么知道这个是完整的，还是只是一个分段啊，又怎么把乱序的分段拼起来啊。

这一切都依赖于首部固定部分的下面几位

  1. 标识：占16位，每生成一个数据报（指没分段的时候）就加一，加以区别。然后分段的时候，每一个小段数据报的标识部分是一样的。
  2. 标志：占3位，但是只有两位有用。第2位（DF位）被置为1时，不分段。第三位（MF位），对于分段的数据报，除了最后一个分段为0以外，其余分段都为1
  3. 偏移量：一共13位。用于表示这个数据报在分段后，相对于原数据报起点的相对位置。偏移量的单位是8个字节。也就是说除了最后一个分段，前面的分段的字节数一定是8的倍数。
  4. TTL（生存时间）：占8位，最大值为255。表示数据报所能经过的最大路由数。每当这个数据报经过一个路由，路由器在转发它之前，先把它的TTL减一，再转发它。如果路由器收到了一个TTL已经为0的数据报，则把它丢弃。TTL的设定是为了防止有些无意义的消息一直在绕圈，占用资源。特别的，如果TTL设为1，则说明这个数据报只能在本地局域网里面传送
  5. 协议：占8位，用来表示是那个协议在向IP协议传送数据。ICMP 为 1，IGMP 为 2，TCP 为 6，UDP 为 17，GRE 为 47，ESP 为 50。以便目的主机在收到数据后，知道该把数据上交给哪个协议。
  6. 首部检验和：占16位，作为首部的校验码
  7. 源IP：发送方的IP
  8. 目的IP：目的主机的IP

可选部分

1到40字节，很少使用。
